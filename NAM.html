Node App Manager (NAM) GitHub

This is a node project written in TypeScript, which utlizes Express and a library called Decache.

The motivation for this was simple: I was fed up with using typical node development tools like nodemon, a program which automatically redeploys a node application when it detects a change to a source file. The issue I have with nodemon's approach is that if you need to run many modules or applications locally, it can take a long time to redeploy everything, and all state like database connections have to be rebuilt for even the smallest, unrelated change.

The solution I developed was to write a library that accomplishes what nodemon does, but at the granularity of an arbitrary unit of code. When NAM starts up, it imports a set of modules from a given target directory. It then watches the target directory for changes to source code, and when such a change is detected, it drops the current version of said module that is being hosted, purges node's cache of the old module, then reimports and rehosts the modified module. This solution accomplishes all of this without impacting any of the other modules which are currently being hosted. Modules that are imported are automatically wrapper in an express router if the default export of the target module isn't an express router, so arbitrary middleware can be run and tested independently and seemlessly.
